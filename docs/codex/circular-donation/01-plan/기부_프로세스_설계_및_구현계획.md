# 기부 프로세스 설계 및 구현 계획

> 작성일: 2026-06-01
> 관련 도메인: 순환재고 판매 (CircularSale), ESG 점수 (ScoreEvents)

---

## 1. 목적

- 순환재고 판매 프로세스를 재사용하여 **기부 경로를 최소 추가 작업으로 구현**한다.
- 기부처(수혜처)는 별도 거래처 등록 없이 **수동 텍스트 입력**으로 처리한다.
- 기부로 발생하는 **탄소 감축량 및 기부 실행 점수**가 ESG 나무 키우기에 자동 반영된다.
- 기존 순환재고 판매 기능에 **손상 없이** DONATION 분기만 추가한다.

---

## 2. 범위

- DB 스키마 변경 (마이그레이션 완료)
- BE: Entity, Service, DTO, Repository, ScoreEvents 수정
- FE: SKU 선택 뷰, 등록 뷰, 이력 뷰, 상세 뷰, ESG 점수 뷰 수정

---

## 3. 핵심 설계 원칙

### 3.1 판매 vs 기부 차이점

| 항목 | 판매 (SALE) | 기부 (DONATION) |
|------|------------|----------------|
| 거래처 | CircularBuyer (DB 등록된 거래처) | 수동 입력 텍스트 (donee_name) |
| buyer_id | NOT NULL | NULL |
| 금액 | 단가 × 수량 | 0 |
| 소재구분 | 선택 필수 | 동일 (SKU 소재 기반) |
| 출고 생성 | 동일 | 동일 |
| 재고 차감 | 동일 | 동일 |
| 판매번호 prefix | CSR- | DON- |

### 3.2 ESG 점수 계산

| 점수 항목 | 판매 (SALE) | 기부 (DONATION) |
|----------|------------|----------------|
| saleExecution | 100pt (10kg↑) | 0 |
| carbon | weight × factor | weight × factor (동일) |
| donationExecution | 0 | 100pt (10kg↑) |
| newBuyer | +150pt (신규 거래처) | 0 |
| localPartner | +150pt (지역 파트너) | 0 |

- **탄소 감축 기여(carbon)**: 판매와 기부 동일 산식 → 기존 `carbonSum`에 자동 합산
- **기부 활동 실행(donationExecution)**: 새 카테고리 — EsgTreeScoreView에 5번째 항목 추가

## 4. DB 스키마 변경 (완료)

```sql
-- circular_sale_header: 기부 타입 및 기부처명 추가
ALTER TABLE circular_sale_header
  ADD COLUMN sale_type  ENUM('SALE','DONATION') NOT NULL DEFAULT 'SALE',
  ADD COLUMN donee_name VARCHAR(200) NULL;

-- circular_buyer_transaction: main_material 제거 + buyer_id nullable + 기부 컬럼 추가
ALTER TABLE circular_buyer_transaction
  DROP COLUMN main_material_code,
  DROP COLUMN main_material_ratio,
  MODIFY COLUMN buyer_id BIGINT NULL,
  ADD COLUMN sale_type  ENUM('SALE','DONATION') NOT NULL DEFAULT 'SALE',
  ADD COLUMN donee_name VARCHAR(200) NULL;
```

### 회귀 안전성
- 기존 SALE 데이터: `sale_type DEFAULT 'SALE'`로 자동 채워짐
- `buyer_id nullable` 전환: 기존 NOT NULL 데이터에 영향 없음
- `main_material` 제거: 탄소 계산에서 이미 미사용 (BLEND 고유 factor 사용)

## 7. 회귀 위험 및 체크리스트

### DB 회귀 위험

| 위험 | 대응 |
|------|------|
| 기존 SALE 데이터 오염 | sale_type DEFAULT 'SALE' — 자동 채워짐 |
| buyer_id NULL 전환 | 기존 데이터 NOT NULL 유지 — 영향 없음 |
| 통계 쿼리에 기부 혼입 | WHERE sale_type = 'SALE' 조건 추가로 차단 |

### BE 회귀 체크리스트

- [ ] SALE 판매 생성 API 정상 동작 (buyerCode 필수, CSR- prefix)
- [ ] SALE 판매 목록/상세 조회 정상 동작
- [ ] ESG 점수 SALE 이벤트 saleExecution/carbon/newBuyer/localPartner 계산 동일
- [ ] 판매 통계 API (kpiAggregate, topVendor 등) 수치 변경 없음
- [ ] DONATION 생성 시 DON- prefix, buyer_id=NULL, doneeName 저장 확인
- [ ] DONATION ESG 점수: donationExecution=100, carbon=weight×factor, newBuyer/localPartner=0

### FE 회귀 체크리스트

- [ ] SKU 선택 뷰 기존 판매 필터 동작 정상
- [ ] 판매 등록 3단계 플로우 정상 (Step2 BuyerSection 그대로)
- [ ] 기부 등록 3단계 플로우 동작 (Step2 DoneeSection)
- [ ] 판매 이력 탭 기존 목록 동일
- [ ] 기부 이력 탭 기부 건만 필터링 확인
- [ ] 판매 상세 기존 표시 동일
- [ ] 기부 상세 산업군/금액 숨김 확인
- [ ] ESG 점수 뷰 기부 활동 실행 항목 표시
