# 기부 프로세스 백엔드 구현 완료 보고서

> 작성일: 2026-06-01  
> 작업 범위: 순환재고 판매(CircularSale) 도메인에 기부(DONATION) 타입 추가

---

## 1. 구현 개요

기존 순환재고 판매 프로세스를 최소한의 확장으로 재사용하여 기부 프로세스를 구현.  
핵심 전략: `saleType = "SALE" | "DONATION"` 분기로 단일 코드베이스에서 판매/기부를 모두 처리.

### 설계 원칙
- 판매와 기부는 SKU 선택 → 출고 → ESG 점수 가산까지 **동일한 흐름**
- 차이점은 **거래처(buyer) 대신 기부처명(donee_name) 텍스트 입력**만
- 탄소 감축량 계산: 판매/기부 모두 동일하게 `weight × carbonFactor` (×0.5 미적용)
- 기부 실행 점수(donationExecution): 판매 실행 점수(saleExecution)와 별도 카테고리로 집계

## 4. 최종 프로세스 흐름

### 판매 (SALE)
```
요청: saleType="SALE", buyerCode="CB-001", items=[...]
  → CircularBuyer 조회 (필수)
  → CircularSaleHeader 저장 (saleNo: CSR-YYYYMMDD-NNNNN)
  → WhOutboundHeader 저장 (destinationType=CIRCULAR_BUYER, destinationId=buyer.id)
  → CircularBuyerTransaction 저장 (buyer=엔티티, saleType="SALE")
  → ESG 점수: saleExecution(100) + carbon(weight×factor) + newBuyer/localPartner 보너스
```

### 기부 (DONATION)
```
요청: saleType="DONATION", doneeName="굿윌스토어", items=[...]
  → buyer 조회 없음
  → CircularSaleHeader 저장 (saleNo: DON-YYYYMMDD-NNNNN, buyerId=null)
  → WhOutboundHeader 저장 (destinationType=DONATION, destinationId=null, destinationName="굿윌스토어")
  → CircularBuyerTransaction 저장 (buyer=null, saleType="DONATION", doneeName="굿윌스토어")
  → ESG 점수: donationExecution(100) + carbon(weight×factor)
```

### 탄소 감축량 계산 (판매/기부 동일)
```
carbon = weightKg × carbonFactor (IPCC Eq.5.2 소각 탄소 계수)

소재별 계수:
  COTTON=1.8, WOOL=1.2, CASHMERE=1.3, SILK=1.3, LINEN=1.7
  POLYESTER=2.3, ACRYLIC=2.4, POLYAMIDE=2.5, ELASTANE=2.2, BLEND=2.0
```

## 6. 매출 통계 보호

순환재고 판매 통계 페이지는 기부 데이터가 섞이지 않도록 6개 통계 쿼리 모두에 필터 적용:
```sql
AND t.sale_type = 'SALE'
```
- `kpiAggregate` (총매출, 활성 거래처 수, 소재 종류)
- `topVendor` (매출 1위 거래처)
- `topMaterial` (판매량 1위 소재)
- `aggregateByVendor` (거래처별 집계)
- `aggregateByMaterial` (소재별 집계)
- `aggregateMonthlyRevenue` (월별 수익)

기부는 `total_amount=0`이므로 필터 없이도 매출 합계에 영향은 없으나,  
거래 건수·거래처 수 등 COUNT 집계 왜곡을 방지하기 위해 명시적으로 제외.
