# 매장발주 승인 출고/입고 오케스트레이션 구현내역 (2026-05-09)

## 1. 구현 범위
- 매장 발주 승인 시 동기 오케스트레이션으로 출고/입고를 자동 생성하도록 백엔드 로직을 확장했다.
- Primary/Backup 창고 분할 할당, 멱등 처리, 상태 이력 생성을 반영했다.
- 분할 출고/입고를 위한 DB 마이그레이션 SQL과 엔티티/리포지토리 변경을 함께 반영했다.

## 2. 핵심 반영 사항
- `StoreOrderService.approveInternal(...)`
  - 승인 시 매장 재고 증가 로직 제거
  - `StoreOrderApprovalOrchestrationService` 호출로 출고/입고 생성 위임
- `StoreOrderApprovalOrchestrationService` 신규 추가
  - Primary 재고 우선 예약, 부족분 Backup 예약
  - 합산 재고 부족 시 `STORE_ORDER_APPROVE_INSUFFICIENT_WAREHOUSE_STOCK` 예외
  - 출고 문서 분할 생성 (`source_ref_seq` 사용)
  - 입고 문서 분할 생성 (`outbound_no` 기준 1:1)
  - 최초 상태 이력 생성
    - 출고: `READY_TO_SHIP`
    - 입고: `PENDING_RECEIPT`
  - 멱등 처리
    - 기존 출고 존재 시 연결 입고 존재를 검증하고 재생성 생략
- 조회 파생값 반영
  - `StoreOrderDto`에 `totalInboundCount`, `receivedInboundCount`, `inboundProgress`, `inboundSummaries` 추가
  - 발주 목록/상세 응답에 파생 진행상태 노출

## 3. DB/엔티티 변경
- `wh_outbound_header`
  - `source_ref_seq` 컬럼 추가
  - 유니크 제약 변경: `(source_type, source_ref_no, source_ref_seq)`
- `store_inbound_header`
  - `delivery_group_no` 컬럼 추가
  - `source_ref_no` 유니크 제거
  - `outbound_no` 유니크 추가
- 마이그레이션 파일
  - `src/main/resources/sql/migration-store-order-approve-obib-2026-05-09.sql`

## 4. 재고 정책 반영
- 승인 시점: 출고지 창고 재고를 예약(`reserved`) 처리
- 기존 승인 시 매장 `available` 증가 처리 제거
- 예약 처리 메서드 추가
  - `InventoryService.reserveForOutboundUpTo(...)`
  - `Inventory.reserveUpTo(...)`

## 5. 에러코드 추가
- `BaseResponseStatus.STORE_ORDER_APPROVE_INSUFFICIENT_WAREHOUSE_STOCK (4611)`

## 6. 확인 필요 사항
- 현재 환경에서 `JAVA_HOME` 미설정으로 gradle test를 실행하지 못했다.
- 실제 검증은 DB 마이그레이션 반영 후 통합 테스트(정상/폴백/부족재고/멱등/롤백)로 추가 확인이 필요하다.
