# 매장 입고 회귀 테스트 / 운영 체크리스트

## 1) 권한/범위 체크
- [ ] 매장 사용자 A가 매장 A 입고건만 조회 가능
- [ ] 매장 사용자 A가 매장 B 입고건 조회 시 차단
- [ ] 매장 사용자 A가 매장 B 입고건 확정 시 차단
- [ ] 창고 사용자 토큰으로 매장 입고 확정 요청 시 차단 정책 확인

## 2) 상태 전이 체크
- [ ] `PENDING_RECEIPT -> RECEIVED` 정상 전이
- [ ] `RECEIVED` 재확정 요청 시 실패
- [ ] 연계 출고 상태가 `ARRIVED`가 아니면 입고 확정 실패
- [ ] 연계 출고 상태가 `ARRIVED`면 입고 확정 성공

## 3) 재고 반영 체크
- [ ] 입고 확정 시 매장 `NORMAL` 재고 `quantity` 증가
- [ ] 입고 확정 시 매장 `NORMAL` 재고 `availableQuantity` 증가
- [ ] 재고 row가 없으면 신규 생성 후 증가 반영
- [ ] 동일 입고건 재확정으로 재고가 중복 증가하지 않음

## 4) 이력/감사 체크
- [ ] 입고 확정 시 `store_inbound_status_history`에 `RECEIVED` 1건 적재
- [ ] 이력의 `changedAt`, `changedByMemberId`, `changedByName`, `reason` 값 확인
- [ ] 발주 완료 전이 시 `store_order_status_history`에 `COMPLETED` 이력 적재

## 5) 발주 연계 체크
- [ ] 분할 입고 2건 중 1건 수령 시 발주 상태 `APPROVED` 유지
- [ ] 분할 입고 전건 수령 시 발주 상태 `COMPLETED` 전이
- [ ] 발주 조회 `inboundProgress`가 `NOT_STARTED/PARTIAL/FULL` 규칙대로 반영

## 6) 롤백/원자성 체크
- [ ] 재고 반영 중 예외 발생 시 입고 상태 전이 미반영(롤백)
- [ ] 입고 이력 저장 실패 시 재고 반영까지 롤백
- [ ] 발주 완료 이력 저장 실패 시 입고 상태/재고까지 롤백

## 7) 운영 점검 체크
- [ ] 수동 출고완료 버튼 경로에서 `ARRIVED` 처리 가능 여부 확인
- [ ] 입고확정 API 호출 전 출고 상태 확인(운영 가이드 반영)
- [ ] 배치 승인 재실행 후 중복 출고/입고 생성 방지 확인
- [ ] 장애 복구 시 재처리 단위(발주번호/출고번호/입고번호) 기준 문서화
